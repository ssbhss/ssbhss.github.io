数据传输类型实现

通过定义数据传输对象，可以将后端繁多的数据返回类型封装为一个统一的格式，方便前端接收。数据传输对象通常包括code、data和msg，当然具体情况视具体项目而定。

1. 定义数据传输对象

public class Result {

    int code;

    Object data;

    String msg;

    @Override
    public String toString() {
        return "Result{" +
                "code=" + code +
                ", data=" + data +
                ", msg='" + msg + '\'' +
                '}';
    }

    public int getCode() {
        return code;
    }

    public void setCode(int code) {
        this.code = code;
    }

    public Object getData() {
        return data;
    }

    public void setData(Object data) {
        this.data = data;
    }

    public String getMsg() {
        return msg;
    }

    public void setMsg(String msg) {
        this.msg = msg;
    }

    public Result(int code, Object data) {
        this.code = code;
        this.data = data;
    }

    public Result(int code, Object data, String msg) {
        this.code = code;
        this.data = data;
        this.msg = msg;
    }

}

2. 对Code进行规定

ublic class Code {

    public static final int PUT_OK   =20011;
    public static final int GET_OK    =20021;
    public static final int DELETE_OK =20031;
    public static final int POST_OK   =20041;

    public static final int PUT_ERR   =20010;
    public static final int GET_ERR    =20020;
    public static final int DELETE_ERR =20030;
    public static final int POST_ERR   =20040;
}

3. 将之前的controller层方法返回值全部改为数据传输类型

@RestController
@RequestMapping("/books")
@ResponseBody
public class BookController {

    @Autowired
    BookService bookService;

    @PostMapping
    public Result save(@RequestBody Book book){
        boolean flag = bookService.save(book);
        return new Result(flag ? Code.POST_OK : Code.POST_ERR, flag);
    }

    @PutMapping
    public Result update(@RequestBody Book book){
        boolean flag = bookService.update(book);
        return new Result(flag ? Code.PUT_OK : Code.PUT_ERR, flag);
    }

    @DeleteMapping(path="/{id}")
    public Result deleteById(@PathVariable int id){
        boolean flag = bookService.deleteById(id);
        return new Result(flag ? Code.DELETE_OK : Code.DELETE_ERR, flag);
    }

    @GetMapping(path="/{id}")
    public Result selectById(@PathVariable int id){
        Book book = bookService.selectById(id);
        return new Result(book != null ? Code.GET_OK : Code.GET_ERR, book, book != null ? "" : "获取书籍失败");
    }

    @GetMapping
    public Result selectAll(){
        List<Book> books = bookService.selectAll();
        return new Result(books != null ? Code.GET_OK : Code.GET_ERR, books, books != null ? "" : "获取所有书籍失败");
    }
}

4. 总结

总的来说，定义数据传输类型是很重要的，构造起来也相对简单，但要注意和前端人员的沟通和联试。
